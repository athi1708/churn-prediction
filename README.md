XGBoost + SMOTE + SHAP + FastAPI + a dark-themed SaaS dashboard.

## Quick Start

### 1. Place the dataset
```
churn_project/data/telco.csv
Download from: https://www.kaggle.com/datasets/blastchar/telco-customer-churn

### 2. Install dependencies
```bash
cd backend
pip install -r requirements.txt
```

### 3. Train the model
```bash
# From project root
python training/train.py
```
This will:
- Clean and encode the Telco dataset
- Engineer 10 new features
- Apply SMOTE oversampling
- Run XGBoost with 30-iteration RandomizedSearchCV
- Calibrate the decision threshold via F1 optimization
- Save model artifacts to `backend/model/`

Expected output:
```
AUC-ROC:  ~0.845
F1 Score: ~0.62
Optimal threshold: ~0.38–0.44
```

### 4. Start the API
```bash
cd backend
uvicorn main:app --reload --port 8000
```

API docs: http://127.0.0.1:8000/docs

### 5. Open the dashboard
Open `frontend/index.html` in your browser.

## API Endpoints

| Method | Endpoint              | Description                        |
|--------|-----------------------|------------------------------------|
| POST   | `/predict`            | Single churn prediction (18 feats) |
| POST   | `/batch_predict`      | Batch predict up to 200 customers  |
| GET    | `/customers`          | Full customer list from CSV        |
| GET    | `/overview`           | Summary stats                      |
| GET    | `/segments`           | Contract-based segments            |
| GET    | `/churn_trend`        | Churn rate by tenure bracket       |
| GET    | `/model/info`         | Model metadata + AUC/F1            |
| GET    | `/monitoring/stats`   | Prediction log + drift detection   |
| DELETE | `/monitoring/reset`   | Clear prediction log               |

## Key Improvements Over v1

| Area              | v1                          | v2                                        |
|-------------------|-----------------------------|-------------------------------------------|
| Model             | RandomForest (default)      | XGBoost + SMOTE + hyperparameter tuning   |
| Threshold         | 0.5 (default)               | Calibrated via F1 optimization (~0.40)    |
| Preprocessing     | Basic encoding + dropna     | Full pipeline + smart TotalCharges impute |
| Feature Eng.      | None                        | 10 derived features (AutoPay, M2MNoSec…)  |
| Insights          | Generic string matching     | Rule-engine with business context         |
| Confidence        | Threshold-based             | SHAP agreement + boundary distance        |
| API structure     | Flat main.py                | Routers + services + Pydantic schemas     |
| Monitoring        | None                        | Prediction log + drift detection          |
| Frontend          | Charts only                 | Risk tiers + confidence card + ROI panel  |
